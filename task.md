# 追加実装タスク

## 前提

- 現行の論文収集、分類、要約、日次/週次配信のコア機能は維持する
- 追加開発はサブスク型サービス化に必要な機能に限定する
- 主な追加対象は会員管理、契約管理、決済、ユーザー別配信

## 分類

- `A`: MVP必須
- `B`: MVP後でもよい
- `C`: リリース前の運用・品質対応

## タスク一覧

### A. MVP必須

#### 1. 会員登録・認証

- [ ] 会員登録画面を作成する
- [ ] 登録項目を `氏名 / メールアドレス / パスワード` に定義する
- [ ] ログイン機能を実装する
- [ ] パスワードをハッシュ化して保存する
- [ ] ログイン状態を維持する仕組みを実装する
- [ ] ログアウト機能を実装する

#### 2. ユーザー情報管理

- [ ] ユーザーごとの基本情報を保存する
- [ ] ユーザーが自分の登録情報を確認できるマイページを作成する
- [ ] アカウント状態 `有効 / 停止` を管理できるようにする

#### 3. 監視条件のユーザー別管理

- [ ] ユーザーごとに監視キーワードを登録できるようにする
- [ ] キーワードの追加、編集、削除をできるようにする
- [ ] ユーザーごとに専門分野を設定できるようにする
- [ ] ユーザーごとに配信頻度 `日次 / 週次` を設定できるようにする
- [ ] ユーザーごとに配信先メールアドレスを設定できるようにする
- [ ] 監視条件の有効/無効を切り替えられるようにする

#### 4. 配信対象制御

- [ ] 固定宛先配信をやめて、ユーザーごとの宛先へ配信できるようにする
- [ ] 有効契約ユーザーだけを配信対象にする
- [ ] 停止中、解約済み、支払失敗のユーザーを配信対象から除外する
- [ ] 配信成功/失敗を記録できるようにする

#### 5. 契約管理

- [ ] 月額契約の開始処理を実装する
- [ ] 契約状態 `契約中 / 解約済み / 支払失敗 / 停止` を管理できるようにする
- [ ] 契約開始日、次回更新日を保持できるようにする
- [ ] 解約処理を実装する

#### 6. 決済連携

- [ ] クレジットカード決済を導入する
- [ ] 初回決済成功で利用開始するフローを実装する
- [ ] 定期課金の更新処理を実装する
- [ ] 支払失敗時の状態遷移を実装する
- [ ] 決済イベントを契約状態に反映する

#### 7. 管理者機能

- [ ] 管理者ログイン機能を実装する
- [ ] 管理者向けユーザー一覧画面を作成する
- [ ] 契約状況一覧を確認できるようにする
- [ ] ユーザーごとの監視条件を確認できるようにする
- [ ] 必要に応じてユーザー停止をできるようにする

#### 8. データ設計

- [ ] ユーザー情報を保存するデータ構造を追加する
- [ ] 契約情報を保存するデータ構造を追加する
- [ ] 監視条件を保存するデータ構造を追加する
- [ ] 配信履歴を保存するデータ構造を追加する
- [ ] 既存のレポート保存処理と新規データ管理の責務を分離する

### B. MVP後でもよい

#### 9. 追加改善

- [ ] メールアドレス変更機能を実装する
- [ ] 同一ユーザーへの重複配信を防ぐ仕組みを強化する
- [ ] 解約後の扱い `即停止 or 更新日まで有効` を仕様確定して柔軟に切り替えられるようにする
- [ ] 配信履歴を管理画面で詳細に確認できるようにする

### C. リリース前の運用・品質対応

#### 10. セキュリティ・運用

- [ ] 管理画面と一般ユーザー画面のアクセス制御を実装する
- [ ] 個人情報を扱う設定値を安全に管理する
- [ ] 必要な環境変数や秘密情報の管理方法を整理する
- [ ] 障害時に配信停止や契約状態を確認できる運用手順を整備する

#### 11. テスト

- [ ] 会員登録とログインのテストを追加する
- [ ] 監視条件登録のテストを追加する
- [ ] 契約状態による配信制御のテストを追加する
- [ ] 決済イベント反映のテストを追加する
- [ ] 管理画面の主要操作のテストを追加する

## 実装優先順位

1. Aの `データ設計`
2. Aの `会員登録・認証`
3. Aの `監視条件のユーザー別管理`
4. Aの `契約管理`
5. Aの `決済連携`
6. Aの `配信対象制御`
7. Aの `管理者機能`
8. Cの `セキュリティ・テスト`
9. Bの `追加改善`

## 依頼主説明用の要約

- 現行システムの本体機能はそのまま使う
- MVPで追加するのは `会員登録 / ユーザー別設定 / 月額課金 / 契約中ユーザーへの個別配信 / 管理画面`
- その後に `細かい改善` と `運用強化` を行う

## 補足

- 現行システムの価値は論文監視と要約配信のコア処理にある
- そのため追加開発は検索ロジックの改修よりも、SaaS運用に必要な外側機能を優先する
